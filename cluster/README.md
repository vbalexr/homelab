# cluster

Per-cluster configurations and FluxCD bootstrap manifests.

Each subdirectory is named after the cluster it manages and contains:

- `flux-system/` — FluxCD system manifests generated by `flux bootstrap`. Do **not** edit these manually.
- `infrastructure/` — Cluster-level infrastructure configuration (ingress controllers, cert-manager, storage classes, etc.).

## Directory Layout

```
cluster/
└── <cluster-name>/
    ├── flux-system/        # Auto-generated by FluxCD bootstrap
    └── infrastructure/     # Cluster infrastructure Kustomizations
```

## Bootstrapping a New Cluster

```bash
flux bootstrap github \
  --owner=<github-owner> \
  --repository=homelab \
  --branch=main \
  --path=cluster/<cluster-name> \
  --personal
```
